F-001 · 登录仅凭 device_id 认证 → 公钥挑战-响应

已修复原 严重 · CVSS 9.8
位置
routes/auth.rs:424, 473, 520, 553 · 迁移 020_*.sql
验证
独立评审 CONFIRMED-FIXED；api-server 编译 + 测试通过
根因

login 仅执行 SELECT id FROM users WHERE device_id = $1 即签发令牌；device_id 非秘密（进日志、随请求头发送），等同于无认证。

修复方案 —— secp256k1 公钥挑战-响应
  1. 新增公开端点 POST /api/v1/auth/challenge：按 device_id 生成 32 字节随机 nonce 存入 login_challenges（5 分钟过期），返回 {challenge}。对未知设备同样返回挑战以防枚举。
  2. 重写 login：请求体改为 {device_id, challenge, signature}，原子化消费挑战并验签。
  3. register 以 COALESCE 持久化 public_key，否则用户无法登录。
// routes/auth.rs:520 — 原子消费，防重放/竞态
"UPDATE login_challenges SET used = TRUE
 WHERE id = $1 AND NOT used AND expires_at > NOW() RETURNING nonce"

// routes/auth.rs:553 — 用存储的公钥对 nonce 摘要验签
if verifying_key.verify_prehash(digest.as_slice(), &signature).is_err() {
    return Err(StatusCode::UNAUTHORIZED);
}

安全性保证：public_key 为 NULL → 拒绝；挑战不存在/过期/已用 → 拒绝；nonce 32 字节随机且单次使用；并发登录竞争同一行仅一胜。

  • 连带打通加密恢复读路径：recovery_service.dart / key_health_service.dart / recovery_view.dart / keys_view.dart 现要求密码并经解密校验。

F-004 · 服务器对任意哈希盲签 → 哈希绑定 + 冻结 + 策略校验

已修复原 严重 · CVSS 8.8
位置
services/mpc_participant/mod.rs:815, 857 及签名路径
验证
评审从 INCOMPLETE → 注释据实修正后一致；编译通过
根因

process_sign_message 直接对客户端任意 32 字节 msg_hash 协同签名，不重派生、不查冻结/策略。

修复方案 —— 要求 raw_tx 并重派生哈希比对
// (a) 重派生哈希并比对 —— raw_tx 缺失即硬失败，无 fall-through
let raw_tx = self.extract_raw_tx(payload)?;          // mod.rs:815
let derived = alloy_primitives::keccak256(&raw_tx);
if derived.as_slice() != msg_hash.as_slice() {
    return Err(format!("tx hash mismatch ... session {}", session_id));
}
// (b) 签名时复查钱包冻结（非仅创建时，避免竞态）
if s == "frozen" { return Err(...refusing to sign...); }
// (c) 查询用户策略行，single_limit_usd <= 0 视为显式封禁
self.enforce_sign_policy(user_id, &raw_tx).await?;   // mod.rs:857

作用域诚实说明（按评审回修）。 MpcParticipant 仅持有 DB 句柄，无价格预言机/HTTP 客户端，因此 定量 USD 限额不在此处执行——它在 routes/tx.rs 广播前执行。此处保证：①签名严格绑定到服务器已见的交易（无法对隐藏摘要取签名）、②冻结复查、③策略行被消费。代码注释已据实修正，不再声称在此做 USD 价值限额。

F-005 · 自研 Paillier 范围证明不健全 → 收紧边界 + 双侧检查 + 小因子检测

已修复原 严重 · CVSS 9.1
位置
crypto/paillier_proof.rs:64, 162, 355
验证
独立评审 CONFIRMED-FIXED（边界数学可证健全，无 off-by-one）；全部 paillier 测试通过
根因

接受界 s1 > 3·q³ 相对诚实响应（~q²）过松，留 q 量级作弊窗口；无下界；模数证明未确立结构。

修复方案 —— 在现有实现上收紧至紧界（采纳方案，不换库）
// prove: 掩码范围由 [0, 2·q³) 收紧为 [0, q³)        (:64)
// verify: 紧接受界（拒绝 >= q³ + q²）               (:162)
//   诚实最大值 = (q³−1)+(q−1)² = q³+q²−2q < q³+q²，
//   与接受界间隙为 O(1)，不再留 q 量级作弊空间。
//   s1 为无符号 BigUint，隐含下界 0，构成正确的双侧范围。
  • 模数证明 verify（:355）新增对前 100 个小素数的 试除小因子检测，并据实注释：这非完整 Paillier-Blum 证明。
  • 新增测试 test_paillier_range_proof_rejects_oversized_s1、test_paillier_modulus_proof_small_factor_rejected。

CONFIRMED-FIXED
根因
